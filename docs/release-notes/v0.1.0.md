# AgentFence v0.1.0

Release theme: local shell permissions and audit logs for AI coding agents.

## Highlights

- Guard shell commands launched through `agentfence run` and the line-oriented `agentfence shell`.
- Classify command risk, including nested shell execution, encoded PowerShell, repository rewrites, remote install pipelines, package publishing, and infrastructure apply/destroy commands.
- Enforce JSON policy decisions for shell, filesystem checks, network domains, skills, MCP servers, MCP tools, MCP resources, and MCP prompts.
- Route MCP stdio and HTTP/batch JSON-RPC calls through AgentFence, including list filtering, rate limits, ask-mode handling, audit events, and argument inspection.
- Upgrade explicitly allowed MCP calls to `ask` when arguments contain secret-looking values, sensitive paths, production/release context, or high-impact tool names.
- Store local SQLite audit events with redaction for common secret shapes.
- Provide a local daemon and desktop control plane for approvals, audit review, policy editing, policy suggestions, simulation, bundles, and exports.
- Ship integration profiles for Codex, Claude Code, Cursor-style agents, and generic MCP clients.
- Publish static project documentation to GitHub Pages.

## Artifacts

CLI and daemon archives:

- `agentfence-windows-x64.zip`
- `agentfence-macos-arm64.zip`
- `agentfence-linux-x64.zip`

Each archive includes:

- `agentfence`
- `agentfenced`
- `agentfence.policy.json`
- `README.md`
- `install.ps1`
- `install.sh`
- A matching `.checksums.json` manifest with SHA256 metadata.

Desktop bundles are produced by the Tauri release job for Windows, macOS, and Linux when supported by the runner.

## Install

Windows:

```powershell
Expand-Archive .\agentfence-windows-x64.zip
cd .\agentfence-windows-x64
.\install.ps1
agentfence policy validate agentfence.policy.json
```

macOS/Linux:

```bash
unzip agentfence-linux-x64.zip
cd agentfence-linux-x64
./install.sh
agentfence policy validate agentfence.policy.json
```

## Current Security Boundary

- Shell enforcement applies to commands launched through AgentFence wrappers.
- MCP enforcement applies to clients routed through `agentfence mcp proxy` or `agentfence mcp http-proxy`.
- Filesystem, network, and skill checks are deterministic policy decisions exposed by CLI and daemon APIs.
- Full PTY interception, kernel-level filesystem interception, and full network proxy/VPN enforcement are not part of v0.1.0.
